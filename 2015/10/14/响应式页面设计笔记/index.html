响应式页面设计笔记

By Hidden Feng

发表于 2015-10-14

前言

偶然的机会遇到了要做一个能适配移动设备的前端页面，所以就有了本文的横空出世，基于我之前做过一些移动设备的前端开发，故希望可以整理一下响应式页面的设计以及适配移动设备的异同，我喜欢条理清晰，目的明确的文章，然而我的语言表达并不能做到言简意赅，所以表述要是有所啰嗦，请提出。我会陆续证据经验进行文章的增删改。

修改1：上面的表述只能说明是孤陋寡闻，以为会响应式就已经会移动开发中的前端开发，其实并不为然。响应式网页设计只是移动优先的跨终端web中的其中给一个手段，所以还不算做个移动开发，只能算是做过响应式开发。

本文将从以下几个方面进行笔记：

  • 响应式的定义以及提出由来
  • 响应式页面的基本要领
  • 移动适配的要点
  • 响应式页面以及移动适配的共同点


一、响应式页面定义 Responsive Page

百度百科上面的解释已经比较全面了，最喜欢的一句话就是：响应式设计更像是一种对于设计的全新思维模式。

维基百科上面的记录了一种 渐进增强 的思路，感觉与百度百科所记录的思路不一样， 百度百科 的比较常规和正式以及详细，然而 维基百科 上面的描述的和百度百科的没有太多的重复却做到了扩展我们视野的目的，不过维基百科上面出现了很多新的名词，有待深究。

二、相应式页面的基本要领

这部分在 响应式网页设计简单入门 中已经阐述的比较详细了，从布局,加载样式表，字体，图片都已经做了基本的简单介绍，虽然并没有很详尽，不过作为入门已经做的很好了。

允许网页宽度自适应 —— 修改

这上面的viewport是html5新添加的，每一个属性都是有意义的，当然，对于自适应来说，只要添加一句基本就可以了。
<meta name="viewport" content="width=device-width,initial-scale=1.0">，已经可以做到网页宽度自适应了。

据说主流的浏览器都支持这个设置，除了（IE6、7、8），网上面有不少这方面的介绍，不过感觉都是没有实践过直接复制别人的贴的，在这里我也没有自己实践过，因为没有IE6、7、8做测试，所以没有做，不过 这篇文章 已经表述的比较清楚了，有条件的时候再 深究 一番。

不使用绝对宽度

由于屏幕宽度要调整布局，所以不能使用绝对宽度的元素，只能使用百分比像素，在移动端开发的时候也是尽可能使用百分比像素的，这样的话，可以不用在媒体查询的时候每个都写一个宽度，貌似目前流行的都是这样写，不过不知道还有没有更加好的解决方式。不过现在基本写代码的时候都是高度自适应以及宽度百分比的吧。

相对字体大小

字体大小的单位有px、em、rem；要使字体大小缩放的更加具体一些的最好使用相对大小，css中比较常用的指定字体相对大小的单位有百分比，em以及css3新增的rem。

响应式网页设计简单入门 这篇文章对于字体缩放的表述还是很详细的，这里就不重复了，不过至于哪一种方式更加适合还是实践之后才会比较清楚。

参考与扩展阅读


  1. Responsive Testing —— 这个工具可以预览所设计的页面再不同设备上的效果，之前直接使用chrome自带的测试工具，虽然可以边调试边预览，不过觉得预览的效果不够强大，现在这个工具可以，所以相互使用还是不错的。

  2. css响应式设计 —— 这个一个关于响应式的图片处理的分析文章，上面提到的技术我觉得不仅可以适应于响应式，还可以使用与webapp开发，因为现在很过会使用平板还打开webapp，所以对于图片的处理还是可以接受的。（至于这个是不是必要以及主流的有待深究，因为现在平板的尺寸更多，是直接使用pc的大小就可以还是要另外适配呢？这也是一个问题）

  3. 响应式网页设计简单入门 —— 这是一篇比较详细的文档，练手的可以参考。

  4. 自适应网页设计（Responsive Web Design）
