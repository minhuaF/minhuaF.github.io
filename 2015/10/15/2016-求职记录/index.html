2016 求职之路

By Hidden Feng

发表于 2015-10-15

前言

已经最后一年了，时间过的真的很快，现在的状态就是不知道自己要做什么，该做什么，能做的之后不要犹豫，不要浪费这一秒钟，从没有试过像现在这样的不安，一直都是总是怕自己会出错的状态，所以很多时候以为可以做一个十全十美的决定，其实没有谁会有这个能力。这几天过的有点虚，所以希望可以通过编写成长历程来记录每一次的感受，不要让自己走偏了。



20151015 华工酒店 欢聚时代（多玩YY）

今天在大学城华工酒店进行的是欢聚时代的面试，应该是所有岗位的面试，没有赶上网申和笔试就只好半面了，带了简历过去，负责人说今天不安排半面的面试，估计没有机会了吧，坐在那里的时候其实可以看到听到蛮多的信息的，看到有一个女生，应该是刚面试“产品策划”出来的，和朋友说面试经过时，可以明显地感觉到她的自信以及面试的过程是那样的享受，她说HR直接就让她进行下一轮的面试了，有时候在想为什么别人就可以那样的游刃有余地面对面试呢？我的自信去哪里了，有时候连大声说话的底气都感觉不行，不是不肯定自己，是自己的努力达不到自己的预期，只能说明自己还不够努力，其实还有很多想做的还没有做的，这是不是就是我的不自信所在，我需要怎么做？!!当初离开“梦聚”时没有解决的问题我是否在解决？?已经模糊了，我自己都不知道。



20151025 广州市天河区天河路365号天俊阁1202室 广州多迪网络科技有限公司

今天在智联招聘上投了简历之后，多迪就通知明天去面试了，本来还是比较开心的，不过看到是一个教育培训机构，有点怕，不过准备面试的过程还是可以收获很多的，希望不要让我来做笔试就好

任职要求：
1、大专以上学历，有设计基础者优先；
2、熟练XHTML/CSS、HTML5、CSS3技术；
3、熟练使用Photoshop，Dreamweaver等网页设计软件,能根据Photoshop效果图来切图；
4、对javascript有较为全面的了解，熟练使用JS完成动画效果、jQuery，jQuery Mobile等前端框架；
5、有手机APP前端开发经验；
6、能解决各大浏览器之间的兼容性问题，包IE6及以上版本，并且有开发自适应屏幕的经验；
7、具备团队合作精神，有较强的沟通及协调能力.

面试前准备

自适应屏幕 —— 有开发项目（汉字识字数测试系统） 一般不另外编写css文件，而是直接在css中，在需要使用CSS媒体查询的地方会直接在对应的css代码段下面直接添加，其他的尽量使用百分比，当然，这是针对布局比较简单的网页才可以使用；主要的是媒体查询规则的使用，即可选择屏幕大小范围，还可以选择横竖屏；这也是其中一种跨终端开发的常用手段（响应式、多站点、多模板、多平台）； —— 使用 CSS 媒体查询创建响应式网站

兼容性问题 这里引出一个 盒模型 的认识：主要是使用标准W3C盒子模型，当然，也是由于存在IE盒子模型，使得会出现一系列的边距IE兼容问题，解决方法一般就是使用display、overflow等属性元素的属性；IE的兼容问题可以说并没有特别处理过，只是有一般的认识，在写代码的时候进行用比较常用的结构或属性，不过这里有一篇文章不错：IE6兼容性问题及IE6常见bug详细汇总

手机APP前端开发 这一块，真的不好说，我做的项目应该只是算一个响应式的移动端页面，不算手机APP前端开发吧，先看看书，那就应该说经验没有，这边使用的开发技术还没有接触过，制作的只是静态页面；

根据Photoshop效果图来切图 切图这块，今天中午看了一下，只是了解了这是怎么回事，技术并不是很熟悉，不过估计是需要根据psd把对应的图片切出来，所以需要吧，不过觉得除了要切小图，其他的直接用代码写会比较快；

HTML5 对于这一块，目前状态比较模糊，编写的代码文档结构中大部分是使用html5，但并不全是，还有部分的div布局；认识比较深刻的就是语义化标签的使用：figure\figcaption\header\aside\footer\address\article\section等等，有利于文档结构;INput中type的设置，还提供的验证功能；头文件中的link、script的引用；新的Doctype引用；音频audio和视频video的引用也只要一个标签就可以，设置属性可以控制对应的变量。插件和API还没有接触很多，并没有实践经验；

CSS3 提供的比较多的动画属性，经过动画(animation)、角度转换（Transform）、3D效果等属性的排列组合可以作出不一样的动画，减少js的使用，也可以制作各种漂亮的按钮，icon等；不过只有ie9及以上浏览器才能很好的支持css3；CSS3较之前的css版本有点在于可以直接用新的属性制作出色的效果，替换css2中要使用图片来展示的效果；可以说一开始就基本接触的就是CSS3，而且对于CSS还没有深入研究，故深层的对比还没有研究好；

jquery 这个的话对基本的选择器、过滤选择器（根据属性或者是：first之类的）、表单选择器（根据input标签的属性值进行条件查找）、操作DOM元素的方法（对css属性，元素的增删改）、Ajax（异步以及数据json的使用）、插件的使用（form、validate、jqzoom、cookie和session的扩展还是比较了解的）

Javascript 对Javascript的了解，就知道可以用于表单验证、cookie创建，动画的制作，对浏览器时间做出响应，还可以与后台服务器进行数据之间的传送；现在要了解更多：

面试后感言

好吧，算是第一次社招面试吧，然后就整个人都不好了，社招与校招的区别真的很大，校招的看个人的综合素质的，有项目经验的话是加分项，没有经验也不要紧，重要的是开发相关的经历；然而校招的眼里只有技术……

面试官一过来就直接问技术相关的问题，主要是看项目做的怎么样，估计是没有笔试，所以就直接看作品判断能力吧，结果就死在那里了，并不是口头回答问题，而是直接看着项目，看着代码问问题，好吧，项目的很多代码都已经不太记得了，记得面试官大概的问题如下：

  • 看简历上说写着“与后台开发人员进行接口数据对接”，讲述一下对接了哪一些数据，怎么对接的？
  • 还有没有其他的项目?
  • 有没有不用前端框架直接手写的代码？
  • jquery这块熟悉程度？
  • Ajax在项目中哪里用到？
  • 看你代码上用了旋转，效果中哪里用了旋转？哪里用了动画？

只想说一句，把代码重温一下再面试的话底气会比较足一下，不过项目经验这方面也是要加强的，明天还要去面试，想想，这样的跑有没有意义呢？



20151028 广州市天河区科韵路16号广州信息港C1栋十楼1001 水晶球

这个面试的过程其实还算可以，就是一个发现问题的过程，面试官张先生还是比较温和且耐心的，虽然知道结局会不怎么样，不过其实还是蛮开心的，开心自己可以坚持去面试，加油哦~！

面试过程

  • 笔试（笔试题目拿回来了，哈哈）
  • 与张先生聊了一段时间，主要是他提问我回答，过程还好啦，不过就是自己的知识认识的太弱了。
    • 一般的玩网页布局怎么实现；
    • 微信端的页面怎么布局；
    • css的优先级；
    • 《前端试题》上面的问题；
    • IE与火狐与谷歌之间的兼容怎么处理；
    • bootstrap框架的使用；
    • jquery的使用；
    • 主要课程的相关信息：最熟悉哪些课程，举例说明；
    • 项目中的SSH框架的理解；
    • html5标签的使用；
    • 数据结构中的典型例子：划分1024个苹果，排列组合；
    • UML中比较典型的图有哪些；
    • ajax请求的时候get和post方式的区别；
    • alert（1&&2）结果是什么；
    • bootstrap比较大，首页加载的时候会很慢，要怎么处理；
    • id以及class的命名规则；

面试后感言

  • 其中最大的感受就是现在的时期很尴尬，不知道是该学习课程知识好还是学习前端的知识比较好，不过感觉就是都要准备。课程的相关知识，就是全部都要懂个大概，然后有某两门要比较精；至于项目方面也要这样，其他要普及，不过还是得要有一两个是比较精通的才可以。
  • 还是觉得自己的专业知识比较弱呀；

20151028 广州市天河区林和西城建大夏 易方达

这个面试应该是面试以来最被重视的一个面试了，不愧是有大公司的风范。

至于易方达的面试官，其实真的很好，从而可以体现一家企业对人才的重视以及这家企业的文化。首先面试的是一个姐姐，好吧，没有记得这么称呼，这并不是技术面，只是对我个人情况的一个了解，我们谈的还是蛮愉快的，主要的成绩，学习情况的了解以及介绍易方达的业务，最尴尬的可能就是问到薪资问题吧，实话就是真的没有要求，但是貌似这个对于企业来书还是必要的，这个问题的确不知道怎么回应，说的5K也是没有什么特别衡量过的。

技术面的时候来了一位很温柔的技术老大，给我的感觉就是很温和，很有耐心，说话的声音也是很轻的，当时就在想，要是能跟着这样的导师一定会很学到很多东西，然后相处也会很不错。

面试后感言
被重视的表现在于面试的时候，简历是不需要另外递交的，面试官会提前打印好，首先会对面试者有一个初步的了解，在面试中也不会常常断片，然后可以比较好地沟通，就目前觉得，这种情况的话不会觉得自己是一个工具，有些公司进门就问你，你的作品有哪些，这种效果会不会做，这种是怎么实现的…等等，虽然可以明白对于技术人员来说，技术很重要，但是，这钟面试只能是体现个人以及公司的综合素质和形象的时候。谈话可以随意，但是起码的尊重还是需要的，即使我的技术不过关，但是既然您邀请我过来面试就得然大家都比较愉快（不是自己申请，直接邀请面试者的），要是前提就已经觉得不合适的话就没有必要浪费大家的时间了，就得先说明情况。我觉得这是最起码的。当然，我入世未深，职场规则不了解。

面试技巧方面还是需要总结，每一种的回答要组织一下语言，随机发挥的虽然会比较多，但是还是要有自己的标准，以及原则。像薪资、加班怎么看，这一类的也是要准备的。还有就是自己的作品的实现思路以及代码编写方面的感悟。

其实面试的时候更多的也是看个人想法以及个人感觉，也可以说是个人经验总结吧，只要对于问题有一定的认识就可以有比较好的应答能力，这个是必须的，然而必须的还是需要结合前期的准备的。
