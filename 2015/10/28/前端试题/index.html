HTML+CSS

1、前端页面由哪三层构成，分别是什么？作用是什么？

结构层 HTML或XHTML等标记性语言 创建了网页的结构层。结构层体现了网页有哪些标签，有哪些内容，标签对内容的语义含义作出描述。

表示层 CSS样式表创建了网页的表现层，表现层就是定义网页文档的外观视觉如何展现的。

行为层

Javascript创建了网页的行为层。行为层定义了网页中的元素具有怎样的行为动作。

2、xhtml与html有什么区别

xhtml(可扩展超文本标记语言)是XHTML 4.01与XML1.0的杂交。html（超文本标记语言），即指页面内可以包含图片、链接，甚至是音乐程序等非文字元素。两者之间的区别主要在两方面：一功能上，XHTML可兼容各大浏览器、手机以及PDA，并且浏览器可以快速正确地编译网页；二是书写规范上，XHTML的语法较为严谨。

4、行内元素有哪些？块级元素有哪些？CSS的盒模型？

行内元素 a b br code i img input label small strong textarea

块级元素 blockquote center div dl ol ul form h1 h2 h3 h4 h5 h6 hr menu p pre table

行级元素与块级元素的区别

  • 块级元素会独占一行，其宽度自动填满其父元素宽度;行内元素不会独占一行，相邻的行内元素会排列在同一行里，知道一行排不下，才会换行，其宽度随元素的内容而变化;
  • 块级元素可以设置 width, height属性，行内元素设置width, height无效;【注意：块级元素即使设置了宽度，仍然是独占一行的】
  • 块级元素可以设置margin 和 padding. 行内元素的水平方向的padding-left,padding-right,margin-left,margin-right 都产生边距效果，但是竖直方向的padding-top,padding-bottom,margin-top,margin-bottom都不会产生边距效果。（水平方向有效，竖直方向无效）

5、CSS引入的方式有哪些？link和@import的区别是？

css的引入有四种：标签内的css；网页内的css；link引用的css；import引用的CSS。

差别1：归属不同（link属于XHTML，@import完全属于CSS提供的一种方式；link还可以定义rel链接属性等）

差别2：加载顺序不同（当一个页面被加载的时候，link引用的CSS会同时被加载，而@import引用的css要等到页面全部被下载完再被加载）

差别3：兼容性（@import是CSS2.1提出，故老的浏览器不支持，只有在IE5以上的才能识别，而link标签都支持）

差别4：使用DOM控制样式时的差别（当使用Javascript控制DOM去改变样式的时候，只能使用link标签，因为@import不是DOM可以控制的）

6、CSS选择符有哪些？哪些属性可以继承？优先级算法如何计算？内联和important哪个优先级高？

CSS选择符

  • 通配选择符（*），便是页面内所有元素的样式。
  • 类型选择符。是指以网页中已有的标签类型作为名称的选择符。如body、div等等。
  • 群组选择符。如果要对一组对象同时进行相同的样式指派，只需要使用“，”，都好，对选择符进行分割即可。
  • 包含选择符。对某对象中的子对象进行样式指派：#header .top{…}。
  • id选择符。#header{…}。
  • class选择符。.titile{…}。
  • 属性选择符。属性选择符是在元素后面加一个中括号，中括号中列出各种属性，或者表达式。属性选择符存在7种具体形式。

CSS属性继承问题

  • 不可继承的：display\margin\border\padding\background\height\position等；
  • 可继承：内联元素（color\letter-spacing\font-size）、块状元素（text-indent\text-align）、列表元素（list-style\list-style-position）、表格元素（border-collapse）

4、相等操作符 == 和 === 的不同

  • 对于Array、Object等高级类型，两者无区别：进行“地址”比较；
  • 对于基础类型：==判断值是否相等，=== 不仅判断值，而且判断类型是否相等；
  • 基础类型与高级类型有区别：==将高级类型转化为基础类型，进行“值”比较，===直接返回false；
  • “==”“===”也会进行隐式转换后进行比较：布尔值、字符串和数值进行比较，转换为数值之后在进行比较；null与undefined是相等的；与NaN判等，返回false；

9、验证输入的是不是整数？

在网上看到有的解决办法是使用 == 而不是 === ，但是我觉得使用 == 是不可取的，因为要是传过来的是一个 string 类型的”123”，使用 == 时还是会返回 true，所以可以考虑使用 === 。下面的两种方式都是从数据类型考虑的，不知道还有没有其他方法。

这里也涉及到了隐式的数据转换知识，不过只对 == 和 === 之间进行讨论。也就是 === 会先做类型检查，而 == 只是单纯的值比较，可以这么理解。

var a = 123;
if(parseInt(a) === a){
    //是整数
}else{
    //不是整数
}；

var a = 123;
if(typeof(a) == "number"){
    //是整数
}else{
    //不是整数
}

10、合并数组的方式？

var a = [2,3,4,5];
var b = ["e","r","t"];

// 方法：concat(); 方法用于连接两个或多个数组。该方法不会改变现有的数组，而仅仅会返回被连接数组的一个副本。
// 局限：使用这种方式连接数组的时候，拼接之后原数组还是会存在，对于小数组可以，但是要是对于大数据数组还是需要优化.
var c = a.concat(b);
    a; //[2,3,4,5]
    b; //["e","r","t"]
    c; //[2,3,4,5,"e","r","t"]


// 方法：Array#push();使用push()方法对数据进行在数组的后面插入。
// 局限：要是插入到数组数据很多，从性能和内存考虑，也是需要优化的。
for(var i = 0;i < b.length;i ++){
     a.push(b[i]);
}
     a; //[2,3,4,5,"e","r","t"]
     b = null;


// 方法： unshift(); 使用unshift()对数组进行前面插入，可以把数据较少的数组插入进数据较多到数组前面。
for(var i = a.length-1 ; i--){
     a.unshift( b[i] );
}
     b; //[2,3,4,5,"e","r","t"]

除了上面提到的方法，还有很多其他的方法，都有不同的优缺点，在这就不再介绍了，平时主要用到的应该也是上面这几个方法。上面的方法主要是参考 JS合并数组的几种方法及优劣比较。
