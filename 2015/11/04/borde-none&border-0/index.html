border:none 与 border:0 的使用区别

By Hidden Feng

发表于 2015-11-04

前言

以前从来不会有这种失望，终于明白机会是留给有准备的人这句话是什么意思了，没有准备很好就不要有什么奢望，不要太嚣张了，保持思路清晰是一件很重要的事情，特别是在面试的时候；

border:none; 与 border:0;

border:none 与 border:0 的区别体现为两点：一是理论上的性能差异，二是浏览器兼容性的差异。

性能差异：

【border:0；】把border设为“0”像素效果等同于borde-width：0，浏览器依然对border-width、border-color进行了渲染，即已经占用了内存值。

【border:none;】把border设为“none”，实际效果鞥同于border-style：none，浏览器解析“none”时将不作出渲染动作，既不会消耗内存值。

兼容性差异：

IE6、IE7中，border为“none”时，标签button、input边框依然存在。

解决办法

  • input，button{border:none;background:none;}
  • input，button{border:0 none;}实践之后证明这个可以兼容到IE6+，不过不知道这样子使用的人多不多；
  • input，button{border:0;}这种方法字符最少，大项目中能节省流量，但是会在渲染的时候消耗内存。

总结

  • 从性能的角度来说更好的是使用 border:none; 但是兼容不好，故要借用 background:none; 来处理兼容； border:0; 性能有待提高，但是能兼容到IE6+；
  • 个人也比较喜欢使用 border:none；故可以选择上面的第二种情况；
